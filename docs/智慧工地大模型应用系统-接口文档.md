# 智慧工地大模型应用系统-接口文档

## 1. 文档说明

本文档描述智慧工地大模型应用系统的后端接口设计，接口内容覆盖系统探活、认证权限、项目管理、文件管理、知识库管理、知识问答、合规审查、报告生成、OCR识别、任务管理和审计日志。

| 状态 | 说明 |
| --- | --- |
| 规划中 | 当前为接口设计，需后续开发实现 |
| 管理端 | 面向平台管理员、项目管理员或运维人员 |
| 业务端 | 面向业务人员、审核人员和项目成员 |
| 内部 | 主要由后端模块、Worker或系统任务调用 |


### 1.1 技术实现说明

- 后端接口由Java + Spring Boot主系统提供。
- 前端由Vue 3 + TypeScript实现，通过REST API调用后端接口。
- 智能算法服务由Python实现，包括大模型调用、RAG检索、OCR识别、文档解析和向量化等能力。
- 前端不直接调用Python算法服务、数据库、对象存储或向量库，所有访问统一经过Java后端接口。
- Java后端与Python算法服务之间通过REST API或内部服务接口集成。

## 2. 通用约定

### 2.1 请求协议

- 协议：HTTP / HTTPS。
- 数据格式：JSON。
- 字符集：UTF-8。
- 文件上传：multipart/form-data。
- 默认服务地址：http://localhost:8080。
- API前缀：/api。

### 2.2 通用请求头

| Header | 必填 | 示例 | 说明 |
| --- | --- | --- | --- |
| Authorization | 是 | Bearer eyJ... | 登录后获取的访问令牌，公开接口除外 |

JWT 鉴权会回查当前用户状态；用户被停用或删除后，旧 token 不再注入认证上下文，后续请求按未登录处理。
| Content-Type | 是 | application/json | JSON接口使用 |
| X-Request-Id | 否 | REQ202607020001 | 请求链路ID，不传由后端生成 |
| Idempotency-Key | 部分接口 | submit-20260702-0001 | 写操作幂等键，生成、重试、提交类接口建议必传 |

### 2.3 统一响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "requestId": "REQ202607020001",
  "timestamp": "2026-07-02T10:00:00+08:00"
}
```

### 2.4 分页响应结构

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "pageNo": 1,
    "pageSize": 20,
```

### 2.5 通用错误码

| code | HTTP状态码 | 含义 | 说明 |
| --- | --- | --- | --- |
| 0 | 200 | 成功 | 请求处理成功 |
| 40000 | 400 | 参数错误 | 参数缺失、格式错误、校验失败 |
| 40100 | 401 | 未认证 | 未登录或Token无效 |
| 40300 | 403 | 无权限 | 当前用户无权访问资源 |
| 40400 | 404 | 资源不存在 | 数据不存在或已删除 |
| 40900 | 409 | 状态冲突 | 当前状态不允许该操作 |
| 42900 | 429 | 请求过频 | 触发限流或重复提交 |
| 50000 | 500 | 系统异常 | 未预期的服务端异常 |
| 50200 | 502 | 外部服务异常 | 大模型、知识库、OCR、对象存储等异常 |
| 50300 | 503 | 服务不可用 | 依赖服务不可用或系统维护 |

## 8. 知识库接口

AI/audit observability rule: `/api/ai/**` external Python calls and audit write APIs must persist trace records with checked affected rows and generated IDs. A missing external-call log or audit log is a conflict for successful operations; if the Python service itself fails, the original error remains primary and the log failure is retained for diagnosis.

Task/Auth write rule: task retry, cancel, worker claim/heartbeat/completion, stage-log insert, outbox insert/delivery-failure updates, user writes, role writes, role-permission links, project-member writes, and login timestamp updates must check affected rows or generated IDs. Zero-row writes return conflict and must not be exposed as success.

## 10. 合规审查接口

### 12.7 OCR内部算法服务约定

OCR识别能力由Qwen VL等多模态模型实现，但Qwen VL调用必须封装在`python-ai-service`中。Java后端`ocr`模块只调用Python智能算法服务，不直接调用`QWEN_VL_ENDPOINT`，不保存Qwen API Key，也不在Java业务层编写模型Prompt。

内部接口：

```text
POST /v1/ocr/recognize
Header: X-AI-Service-Key
```

Java请求Python服务时传入项目ID、OCR记录ID、识别类型、文件临时访问地址、发票类型和自定义字段定义。Python服务负责下载图片临时访问地址，将图片转为`data:image/...;base64,...`后调用Qwen VL，返回统一结构化JSON。这样Qwen云端不需要直接访问本地或内网MinIO临时URL。PDF文件如使用内网临时地址，应由Python侧增加PDF转图片预处理后再调用Qwen VL。

请求示例：

```json
{
  "projectId": 1001,
  "recordId": 30001,
  "ocrType": "CUSTOM",
  "file": {
    "fileId": 4401,
    "fileName": "contract.pdf",
    "contentType": "application/pdf",
    "downloadUrl": "temporary-internal-signed-url"
  },
  "options": {
    "invoiceType": null,
    "customFields": [
      {
        "fieldKey": "partyA",
        "fieldName": "甲方",
        "description": "合同中的甲方名称",
        "required": true,
        "valueType": "TEXT"
      }

  }
}
```


### 13.3 重试任务

- Method：POST
- URL：/api/tasks/{taskId}/retry
- 状态：已实现

说明：仅 `FAILED` 且 `retryCount < maxRetryCount` 的任务允许重试。接口会将任务状态更新为 `QUEUED`，清空错误和取消标记，并写入 `RETRY_REQUESTED` 阶段日志；状态已变化或超过次数时返回冲突错误，不做静默成功。

实现约束：重试成功后写入 `task_outbox`，由可选调度器投递 Redis 队列。`task_outbox` 是任务投递事实源，Redis 投递失败时记录错误、投递次数和下次投递时间。

Worker 约束：后台 Worker 执行业务前必须先领取 `QUEUED` 任务，领取成功后任务进入 `RUNNING`，并写入 `workerId`、`leaseUntil` 和心跳时间。具体业务执行前还必须重新校验项目仍为 `ENABLED`；心跳、成功完成、失败完成都必须校验 worker owner；任务被取消、状态变化或 owner 不匹配时返回冲突错误，不做静默成功。

### 13.4 取消任务

- Method：POST
- URL：/api/tasks/{taskId}/cancel
- 状态：已实现

说明：`PENDING`、`QUEUED`、`RETRYING` 任务会更新为 `CANCELED`；`RUNNING` 任务只写入 `cancelRequested=true`，由 Worker 协作停止；`SUCCESS`、`FAILED`、`CANCELED` 等终态任务返回冲突错误。

### 15.4 接口实现约束

- 所有写操作建议支持 `Idempotency-Key`，避免重复提交任务或重复生成报告。
- 所有项目级接口必须校验当前用户是否拥有目标项目权限。
- 文件下载、预览、报告下载、OCR结果下载必须在生成访问地址前校验数据权限。
- 长耗时接口不得阻塞HTTP请求，应返回业务ID、任务ID和初始状态，由前端轮询任务接口。
- 所有调用Python智能算法服务的接口必须记录外部调用日志，包含服务名、调用类型、请求摘要、响应摘要、耗时、状态和错误信息。
- 数据库问答接口只能执行只读SQL，必须限制最大返回行数、执行超时和敏感字段展示。
- 所有列表接口默认过滤 `deleted = 0`，项目级数据必须包含 `projectId` 查询条件或从路径参数推导项目范围。

## 18. AI Adapter APIs

### 18.1 Model Invoke

`POST /api/ai/model/invoke` accepts `projectId`, `prompt`, `systemPrompt`, `modelName`, `parameters`, and `contextMessages`; returns `answer`, `usage`, and `providerTraceId`.

### 18.2 Agent Invoke

`POST /api/ai/agent/invoke` accepts `projectId`, `goal`, `tools`, `contextMessages`, and `parameters`; returns `result`, `steps`, `followUpQuestions`, and `providerTraceId`.

### 18.3 RAG Search

`POST /api/ai/knowledge/search` accepts `projectId`, `query`, `knowledgeBaseIds`, `libraryTypes`, `topK`, `scoreThreshold`, and `rerankEnabled`; returns source records, snippets, scores, and metadata.

### 18.4 Database Q&A

`POST /api/ai/database/query` accepts `projectId`, `dataSourceId`, `question`, and `context`. The Python service generates SQL and result summaries; Java validates and executes read-only queries against enabled MySQL data sources in the current project.

### 18.5 Routing And Context

`POST /api/ai/route` returns `MODEL`, `KNOWLEDGE`, `DATABASE`, `HYBRID`, or `NEED_MORE_INFO`. `POST /api/ai/context/prepare` returns compact context messages, referenced messages, missing fields, and follow-up questions.

### 18.6 External Call Logs

`GET /api/ai/external-call-logs` supports paged query by `projectId`, `serviceName`, `callType`, and `status`.


## Advanced AI Adapter Update

The AI adapter now includes document indexing, chunking, embedding, vector retrieval, reranking, and a Python tool-calling Agent framework. The Python service supports `LOCAL`, `PGVECTOR`, and `MILVUS` vector providers. Qwen embeddings are used in production through `EMBEDDING_PROVIDER=QWEN`; Qwen rerank is configured through `RERANK_PROVIDER=QWEN`, `QWEN_RERANK_BASE_URL`, and `QWEN_RERANK_MODEL`; `LOCAL_HASH` is reserved for offline tests. Java exposes `/api/ai/knowledge/index` and `/api/ai/knowledge/search`, but Java does not directly access vector databases.

Database Q&A now supports MySQL, PostgreSQL, and Kingbase JDBC drivers while preserving strict read-only SQL validation, query timeout, and maximum-row limits.


PostgreSQL and Kingbase JDBC drivers are enabled for database Q&A execution. PostgreSQL can be integration-tested with `AI_TEST_POSTGRES_JDBC_URL`; Kingbase can be integration-tested with `AI_TEST_KINGBASE_JDBC_URL` when an available Kingbase test instance exists.
